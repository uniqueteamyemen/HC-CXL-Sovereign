# HC-CXL v2.0 — Delivery Package (حزمة التنفيذ)
مالك: د. أبوبكر أحمد عوض
Zone of Trust: `D:\HC-CXL`

## ما في هذه الحزمة (ملخص سريع)
- ملفات جاهزة للرفع إلى GitHub (أوامر PowerShell + bash).
- ملف GitHub Actions CI لتشغيل حاويات Docker والاختبارات.
- سكربت لرفع الملفات إلى Zenodo عبر API (Python).
- سكربت توقيع SHA-256 وحزم الأرشيف.
- خادم خارجي بسيط (Flask) لاستقبال نتائج الاختبارات الخارجية، أرشفتها، وتشغيل محلل بسيط.
- ملف تصدير Workflow لـ n8n لاستقبال webhooks وتحويلها إلى أرشيف أو رفعها إلى Zenodo.
- إرشادات خطوة-بخطوة للحفاظ على السيادة والخصوصية.

**ملاحظة هامة:** لا تقم بتشغيل أي سكربت يرفع بيانات حساسة إلى الخدمات الخارجية دون تهيئة المتغيرات البيئية (TOKENS) ومراجعة المحتوى. كل الأوامر مخصّصة للعمل محليًا في `D:\HC-CXL` أو بيئة آمنة لديك.

---

## أولوية التنفيذ المقترحة (نفّذهم بالترتيب التالي)
1. تشغيل `sign_and_package.py` محليًا لحزم النسخة الحالية (سيحفظ المخرجات في `./artifacts`).
2. فحص CI (`.github/workflows/ci.yml`) وتخصيص الأسرار في GitHub قبل التفعيل.
3. تشغيل الخادم الخارجي `external_server.py` محليًا أو على خادم خاص.
4. تهيئة n8n واستيراد `n8n_workflow.json`.
5. عندما تقرر: استخدم `gh_bootstrap.ps1` أو `bootstrap.sh` لإنشاء المستودع ورفع الملفات (تحتاج gh CLI / git وبيانات اعتماد آمنة).
6. استخدام `zenodo_upload.py` لرفع ملف الأرشيف إلى Zenodo (تحتاج ZENODO_TOKEN).

اقرأ الملفات أدناه للاطّلاع على تفاصيل التنفيذ.
